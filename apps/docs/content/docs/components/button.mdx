import { Button } from "@friday-sandbox/react";

The `Button` triggers an action. It wraps
[React Aria `Button`](https://react-spectrum.adobe.com/react-aria/Button.html),
so it is keyboard accessible and exposes press behaviour through `onPress`.

</Preview>

## States

`isDisabled` blocks interaction; `isPending` shows a spinner for in-flight
actions.

## Props

`Button` also accepts every prop from React Aria `Button`, such as `onPress`,
`isDisabled`, `isPending`, and `type`, and a `className` string.

| Prop            | Type                                                                            | Default     |
| --------------- | ------------------------------------------------------------------------------- | ----------- |
| `color`         | `"primary" \| "secondary" \| "accent" \| "info" \| "success" \| "warning" \| "danger"` | `"primary"` |
| `variant`       | `"solid" \| "subtle" \| "surface" \| "outline" \| "ghost" \| "plain"`           | `"solid"`   |
| `size`          | `"xs" \| "sm" \| "md" \| "lg" \| "xl"`                                          | `"md"`      |
| `isIconOnly`    | `boolean`                                                                       | `false`     |
| `isFullWidth`   | `boolean`                                                                       | `false`     |
| `isRoundedFull` | `boolean`                                                                       | `false`     |
| `className`     | `string`                                                                        | —           |

## Accessibility

- Rendered as a native `<button>` and reachable by keyboard.
- Activates with both <kbd>Enter</kbd> and <kbd>Space</kbd>.
- Use `isDisabled` rather than the native `disabled` attribute so assistive
  technology and focus management stay consistent.
- When a button shows only an icon, set `isIconOnly` and provide an accessible
  label with `aria-label`.
