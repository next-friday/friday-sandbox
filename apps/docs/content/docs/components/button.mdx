## Usage

The `Button` triggers a single action on the current view, such as saving or
submitting. It wraps React Aria `Button`, so it is keyboard accessible and
exposes press behavior through `onPress`. To navigate to another page, use a
link instead.

## Props

`Button` also accepts every prop from React Aria `Button`, such as `onPress`,
`isDisabled`, `isPending`, and `type`, plus a `className` string.

| Prop            | Type                                                                                   | Default     | Description                                              |
| --------------- | -------------------------------------------------------------------------------------- | ----------- | ------------------------------------------------------- |
| `color`         | `"primary" \| "secondary" \| "accent" \| "info" \| "success" \| "warning" \| "danger"` | `"primary"` | Sets the semantic color role.                           |
| `variant`       | `"solid" \| "subtle" \| "surface" \| "outline" \| "ghost" \| "plain"`                  | `"solid"`   | Sets the visual emphasis.                               |
| `size`          | `"xs" \| "sm" \| "md" \| "lg" \| "xl"`                                                 | `"md"`      | Sets the button size.                                   |
| `isIconOnly`    | `boolean`                                                                              | `false`     | Renders a square icon-only button; pair with `aria-label`. |
| `isFullWidth`   | `boolean`                                                                              | `false`     | Stretches the button to the container width.            |
| `isRoundedFull` | `boolean`                                                                              | `false`     | Uses a fully rounded pill shape.                        |
| `className`     | `string`                                                                               | —           | Additional CSS classes to apply.                        |

## Accessibility

- Rendered as a native `<button>` and reachable by keyboard.
- Activates with both <kbd>Enter</kbd> and <kbd>Space</kbd>.
- Use `isDisabled` rather than the native `disabled` attribute so assistive
  technology and focus management stay consistent.
- When a button shows only an icon, set `isIconOnly` and provide an accessible
  label with `aria-label`.

## Styling

`Button` is styled through CSS classes ([source styles](https://github.com/next-friday/friday-sandbox/blob/main/packages/styles/src/components/bases/button.css)), so the look applies to any element that carries them.

### Base class

- `fri-button` — base button styles with the primary color, solid variant, and medium-size defaults.

### Color modifier classes

- `fri-button-primary` · `fri-button-secondary` · `fri-button-accent` · `fri-button-info` · `fri-button-success` · `fri-button-warning` · `fri-button-danger`

### Variant modifier classes

- `fri-button-solid` · `fri-button-subtle` · `fri-button-surface` · `fri-button-outline` · `fri-button-ghost` · `fri-button-plain`

### Size modifier classes

- `fri-button-xs` · `fri-button-sm` · `fri-button-md` · `fri-button-lg` · `fri-button-xl`

### Shape and width modifier classes

- `fri-button-icon-only` — square, icon-only footprint.
- `fri-button-full-width` — stretches to the container width.
- `fri-button-rounded-full` — fully rounded pill shape.

### State selectors

State styling responds to React Aria data attributes: `[data-hovered]`, `[data-pressed]`, `[data-disabled]`, `[data-focus-visible]`, and `[data-pending]`.
