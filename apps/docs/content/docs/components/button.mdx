## Purpose

The `Button` triggers a single action on the current view, such as saving or
submitting. It wraps React Aria `Button`, so it is keyboard accessible and
exposes press behavior through `onPress`.

## When to use

- Trigger a single action on the current view, such as saving, submitting, or
  confirming.
- You need keyboard-accessible press behavior through `onPress`.

## When not to use

- To navigate to another page or view — use a link instead.

## Props

`Button` also accepts every prop from React Aria `Button`, such as `onPress`,
`isDisabled`, `isPending`, and `type`, plus a `className` string.

| Prop            | Default     | Type                                                                     |
| --------------- | ----------- | ----------------------------------------------------------------------- |
| `color`         | `"primary"` | `"primary" \| "secondary" \| "accent" \| "info" \| "success" \| "warning" \| "danger"`<br />Sets the semantic color role. |
| `variant`       | `"solid"`   | `"solid" \| "subtle" \| "surface" \| "outline" \| "ghost" \| "plain"`<br />Sets the visual emphasis. |
| `size`          | `"md"`      | `"xs" \| "sm" \| "md" \| "lg" \| "xl"`<br />Sets the button size. |
| `isIconOnly`    | `false`     | `boolean`<br />Renders a square icon-only button. Pair it with `aria-label`. |
| `isFullWidth`   | `false`     | `boolean`<br />Stretches the button to the container width. |
| `isRoundedFull` | `false`     | `boolean`<br />Uses a fully rounded pill shape. |
| `className`     | —           | `string`<br />Additional CSS classes to apply. |

## Styling

`Button` is styled through CSS classes, so the look applies to any element that carries them.

### Modifier classes

| Class                     | Type     | Description                                          |
| ------------------------- | -------- | --------------------------------------------------- |
| `fri-button`              | Base     | Base styles: primary color, solid variant, md size. |
| `fri-button-primary`      | Color    | Primary color role.                                 |

| `fri-button-accent`       | Color    | Accent color role.                                  |

| `fri-button-success`      | Color    | Success color role.                                 |
| `fri-button-warning`      | Color    | Warning color role.                                 |
| `fri-button-danger`       | Color    | Danger color role.                                  |
| `fri-button-solid`        | Variant  | Solid variant.                                      |

| `fri-button-surface`      | Variant  | Surface variant.                                    |
| `fri-button-outline`      | Variant  | Outline variant.                                    |
| `fri-button-ghost`        | Variant  | Ghost variant.                                      |

### State selectors

State styling responds to these React Aria data attributes:

| Selector               | Applies when                              |
| ---------------------- | ----------------------------------------- |
| `[data-hovered]`       | A pointer is over the button.             |
| `[data-pressed]`       | The button is being pressed or activated. |
| `[data-focus-visible]` | The button is focused by keyboard.        |
| `[data-disabled]`      | The button is disabled.                   |
| `[data-pending]`       | The button is in a pending state.         |

## Accessibility

- Rendered as a native `<button>` and reachable by keyboard.
- Activates with both <kbd>Enter</kbd> and <kbd>Space</kbd>.
- Use `isDisabled` rather than the native `disabled` attribute so assistive
  technology and focus management stay consistent.
- When a button shows only an icon, set `isIconOnly` and provide an accessible
  label with `aria-label`.
