  Accessible React UI components built on react-aria-components and Tailwind CSS
  v4.

Components compose [`react-aria-components`](https://react-spectrum.adobe.com/react-aria/components.html)
for focus, selection, and keyboard behavior, and style with tokens from
`@friday-sandbox/styles`. Sources ship un-bundled — consumers import directly
from `src/`, so tree-shaking is the bundler's job, not ours.

## Install

```sh
pnpm add @friday-sandbox/react @friday-sandbox/styles
```

Peer dependencies: `react@^19`, `react-dom@^19`, `tailwindcss@^4`.

## Usage

Import the stylesheet once at the application root:

```ts
import "@friday-sandbox/styles";
```

Then import components by subpath:

```tsx
import { Button } from "@friday-sandbox/react";

export function Example() {
  return <Button variant="primary">Click me</Button>;
}
```

## Theming

Themes are switched via the `data-theme` attribute on `<html>`:

```html
<html data-theme="light">
  <!-- or "dark" -->
</html>
```

The Storybook toolbar exposes the same switch under the **Theme** control.

## Conventions

- One component per file under `src/<name>.tsx`, named export.
- `"use client"` only when a client API is touched.
- Style with Tailwind utilities + tokens from `@friday-sandbox/styles` — never
  bypass tokens with hardcoded hex values.
- Every component story covers default, hover, focus, disabled, loading, and
  error states.
