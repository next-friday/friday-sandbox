# Tokens

Override any of these in your app's CSS to retheme.

## Intent colors

Eight brand colors. Seven ship distinct light and dark values; only `neutral` is identical in both modes. Components derive readable foreground and hover state from each token.

<div className="mt-4 grid grid-cols-4 gap-3">
  {[
    { intent: "primary", swatch: "bg-primary" },
    { intent: "secondary", swatch: "bg-secondary" },
    { intent: "accent", swatch: "bg-accent" },
    { intent: "neutral", swatch: "bg-neutral" },
    { intent: "info", swatch: "bg-info" },
    { intent: "success", swatch: "bg-success" },
    { intent: "warning", swatch: "bg-warning" },
    { intent: "danger", swatch: "bg-danger" },
  ].map(({ intent, swatch }) => (
    <div key={intent}>
      <div className={`h-12 rounded-action ${swatch}`} />
      <code className="mt-1 block text-xs">--{intent}</code>
    </div>
  ))}
</div>

## Using tokens with Tailwind

A token registered with `@theme inline` resolves to a canonical Tailwind v4 utility. Reach for the alias such as `bg-primary` instead of the arbitrary-var fallback `bg-(--primary)`. Component-local variables such as `--grid-min` are the exception and use the arbitrary-var form.

<div className="mt-4 flex flex-wrap gap-3">
  <span className="rounded-action bg-primary px-4 py-2 text-sm font-medium text-primary-foreground">
    bg-primary
  </span>
  <span className="rounded-action bg-danger px-4 py-2 text-sm font-medium text-danger-foreground">
    bg-danger
  </span>
  <span className="rounded-box bg-muted px-4 py-2 text-sm text-foreground">
    bg-muted
  </span>
</div>

```tsx
<button className="rounded-action bg-primary px-4 py-2 text-primary-foreground">
  Save
</button>
<p className="text-foreground">Body text</p>
<div className="border-danger border" />
```

| Token kind                                  | Utilities                                                                                                                       |
| ------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| Intent (`--primary` … `--danger`)           | `bg-<intent>`, `text-<intent>`, `border-<intent>`, `ring-<intent>`, `fill-<intent>`, `stroke-<intent>`, each with a `-foreground` pair |
| Neutral (`--neutral`)                       | `bg-neutral`, `text-neutral`, and the rest; no `-foreground` token                                                             |
| Radius (`--radius-{scope}`)                 | `rounded-action`, `rounded-field`, `rounded-box`                                                                               |
| Border (`--border`)                         | `border`                                                                                                                       |
| Surface (`--surface`, `--surface-foreground`) | `bg-surface`, `text-surface-foreground`, and the rest                                       |

## Surfaces

Light by default; flipped by `.dark` or `[data-theme="dark"]`. Values shown light / dark.

| Token                  | Role                                                       | Light             | Dark               | Tailwind alias                  |
| ---------------------- | --------------------------------------------------------- | ----------------- | ------------------ | ------------------------------- |
| `--background`         | Body surface                                              | `100% 0 0`        | `18% 0 0`          | `bg-background`                 |
| `--foreground`         | Body text on `--background`                               | `0 0 0`           | `95% 0 0`          | `text-foreground`               |
| `--muted`              | Secondary surface, such as a placeholder                  | `89.2% 0.0051 228.83` | `30% 0.006 228.83` | `bg-muted`                      |
| `--surface`            | Inverted surface, such as a tooltip                       | `20.9% 0 0`       | `95% 0 0`          | `bg-surface`      |
| `--surface-foreground` | Text on `--surface`                                       | `100% 0 0`        | `20.9% 0 0`        | `text-surface-foreground` |
| `--neutral`            | Mid-tone gray, identical in both modes; no foreground pair | `65.47% 0.0208 248.15` | `65.47% 0.0208 248.15` | `bg-neutral`                    |
| `--ring`               | Focus ring color                                          | `57.49% 0.2084 257.52` | `64% 0.18 257.52`  | `ring-ring`                     |

All values are the OKLCH channels passed to `oklch(...)` in the default theme.

## Foreground overrides

Each intent except `neutral` ships a paired `--*-foreground` token that components use for readable text. Override these to retune contrast.

| Token                    | Light       | Dark        |
| ------------------------ | ----------- | ----------- |
| `--primary-foreground`   | `100% 0 0`  | `100% 0 0`  |
| `--secondary-foreground` | `100% 0 0`  | `100% 0 0`  |
| `--accent-foreground`    | `100% 0 0`  | `0 0 0`     |
| `--info-foreground`      | `100% 0 0`  | `100% 0 0`  |
| `--success-foreground`   | `100% 0 0`  | `100% 0 0`  |
| `--warning-foreground`   | `100% 0 0`  | `20.9% 0 0` |
| `--danger-foreground`    | `100% 0 0`  | `100% 0 0`  |

## Sizing scopes

| Scope    | Tokens                             | Component family                         |
| -------- | ---------------------------------- | ---------------------------------------- |
| `action` | `--size-action`, `--radius-action` | button, icon-button, link-button         |
| `field`  | `--size-field`, `--radius-field`   | input, textarea, select, checkbox, radio |
| `box`    | `--size-box`, `--radius-box`       | card, modal, alert, popover              |

### Default sizing

At the default `--size-{scope}` of `0.25rem`, the five sizes resolve to:

| Size | Height         |
| ---- | -------------- |
| `xs` | 1.75rem (28 px) |
| `sm` | 2rem (32 px)    |
| `md` | 2.5rem (40 px)  |
| `lg` | 2.75rem (44 px) |
| `xl` | 3rem (48 px)    |

Override `--size-action` (or `--size-field` / `--size-box`) and every size for that scope rescales together.

`--radius-{scope}` is the reference at `md`. Smaller sizes get proportionally smaller corners, larger sizes get larger ones, so roundness stays balanced from `xs` to `xl`.
